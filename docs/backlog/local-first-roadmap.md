# Local-First Roadmap

Use this backlog to prove the personal learning loop before expanding to Supabase, Vercel, auth, or runtime API calls.

The current strategy is manual-Codex-first:

1. The app records reliable learning evidence locally.
2. The learner exports or copies a tutor packet.
3. Codex manually generates the next drill, lesson, or progress review.
4. Useful content is committed to the repo or stored in skill records.

Runtime AI is out of scope until this manual loop becomes a real bottleneck.

## Expansion Decision Slices

### LT-014 IndexedDB Readiness ADR
- Outcome: decide when to move from localStorage to IndexedDB.
- Why: localStorage is enough for now, but not forever.
- Acceptance:
  - data-size triggers
  - migration approach
  - export/backup implications
  - no implementation unless approved

### LT-015 Runtime AI API ADR
- Outcome: decide whether to ever add a `Generate next drill` button backed by API calls.
- Why: API billing is separate and personal-use manual Codex may be enough.
- Acceptance:
  - model/cost estimate
  - API key handling plan
  - monthly budget and kill switch
  - manual Codex fallback
  - privacy note for learner data

### LT-016 Supabase/Vercel Expansion ADR
- Outcome: decide whether hosted sync/deployment is worth the scope.
- Why: personal local app should not accidentally become a SaaS.
- Acceptance:
  - personal-use trigger
  - family/shared-use trigger
  - auth/profile implications
  - sync conflict strategy
  - rollback path to local-first
